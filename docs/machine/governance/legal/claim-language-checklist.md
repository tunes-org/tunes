# Claim language checklist (ADR-011)

**Status:** Draft — use before map, press, or marketing copy ships  
**Version:** 0.1.0-draft  
**Last-updated:** 2026-07-20

Tick before publishing user-facing claims:

- [ ] Prefer precise acoustic terms (e.g. LAeq,T, ranges, sample counts, quality tier)  
- [ ] “Discomfort” only when referring to **subjective survey** answers  
- [ ] No “acoustic risk”, health harm, or regulatory exposure/compliance claims for crowd data  
- [ ] “Quieter journey” is comparative **within dataset limits**, with uncertainty shown  
- [ ] No implied TfL operation, approval, or endorsement  
- [ ] Survey-grade / not Class 1 stated where users might over-read precision  
- [ ] Sparse sections not presented as definitive rankings  
- [ ] Objective metrics and subjective responses not blended into one unexplained score
